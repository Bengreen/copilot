  QUICK START

1. Read the welcome:
   cat ~/.copilot/rfc-spec-system/START-HERE.md

2. Read the quick start:
   cat ~/.copilot/rfc-spec-system/QUICKSTART.md

3. In your project, set up the structure:
   mkdir -p spec/requirements/{architecture,features,documentation}

4. Copy templates to your project:
   cp ~/.copilot/rfc-spec-system/README.md spec/
   cp ~/.copilot/rfc-spec-system/QUICKSTART.md spec/
   cp ~/.copilot/rfc-spec-system/ARCHITECTURE-TEMPLATE.md spec/ARCHITECTURE.md
   cp ~/.copilot/rfc-spec-system/DOCUMENTATION-ANALYSIS-TEMPLATE.md spec/DOCUMENTATION_ANALYSIS.md
   cp ~/.copilot/rfc-spec-system/REQUIREMENTS-INDEX-TEMPLATE.md spec/requirements/README.md

5. Create first requirement:
   cp ~/.copilot/rfc-spec-system/REQ-TEMPLATE.md spec/requirements/architecture/REQ-A001.md

6. Edit, add to index, and commit!

================================================================================
  KEY CONCEPTS
================================================================================

RFC-Inspired:      Versioned, auditable requirements (like IETF standards)
Global Numbering:  REQ-A001, REQ-F005, REQ-D014 (each number unique)
Modular:           One file per requirement (easy version control)
Unified:           Spec + work items + results in one file

Categories:
  A = Architecture  (system design, patterns)
  F = Features      (functionality, capabilities)
  D = Documentation (guides, specs, docs)

Status:
  ACTIVE    = Currently implemented/in development
  PROPOSED  = Planned for future
  SUPERSEDED = Replaced by newer version
